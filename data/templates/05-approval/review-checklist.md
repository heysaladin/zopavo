## A — Brand Identity Review

### Logo & Mark
- [ ] Logo is vectorised (AI or SVG source)
- [ ] Primary, horizontal, and icon-only versions exist
- [ ] Works on light and dark backgrounds
- [ ] Minimum size tested (legible at 16px height)
- [ ] Clear space defined and respected
- [ ] No copyright or trademark conflicts (basic check)

### Colour
- [ ] Primary and secondary colours defined with HEX, RGB, CMYK values
- [ ] Contrast ratios checked (text on background ≥ 4.5:1 for body)
- [ ] Colour combinations tested for accessibility (colour blindness basic check)

### Typography
- [ ] Font licences confirmed (web, print, commercial use)
- [ ] Font pairing tested across heading sizes (H1–H4)
- [ ] Fallback fonts defined
- [ ] Body text legible at 16px on screen

### Brand Guidelines Document
- [ ] Table of contents present
- [ ] Logo usage rules clearly documented
- [ ] Do/don't examples included
- [ ] Colour values in all formats (HEX, RGB, CMYK, Pantone if needed)
- [ ] Typography scale shown
- [ ] Tone of voice section included
- [ ] Grid/layout principles shown (if applicable)

---

## B — Website / UI Design Review

### Visual Quality
- [ ] Consistent spacing system used (8px grid or defined system)
- [ ] Component states designed (default, hover, active, disabled, error)
- [ ] Icons consistent in style, weight, and size
- [ ] Images are high resolution or properly sourced
- [ ] No lorem ipsum text in designs presented to client

### Brand Alignment
- [ ] Colours match approved brand palette
- [ ] Typography matches brand guidelines
- [ ] Tone of imagery aligns with brand direction
- [ ] Spacing/density matches brand personality (e.g. premium = more white space)

### Responsive Design
- [ ] Desktop breakpoint designed (1440px or 1280px)
- [ ] Tablet breakpoint designed (768px)
- [ ] Mobile breakpoint designed (375px or 390px)
- [ ] No content cut off or overlapping on any breakpoint

### Figma File Hygiene
- [ ] Layers are named (no "Frame 143")
- [ ] Components used from shared library where applicable
- [ ] Pages are labelled clearly
- [ ] Prototype flows linked (if applicable)

---

## C — Development / Build Review

- [ ] Matches approved Figma design (pixel comparison spot-check)
- [ ] Responsive on Chrome, Safari, Firefox (last 2 versions)
- [ ] Mobile tested on iOS Safari and Android Chrome
- [ ] Page load time < 3s on 4G connection (Lighthouse check)
- [ ] No broken links
- [ ] Forms submit correctly and send to correct destination
- [ ] SEO basics: page titles, meta descriptions, OG tags set
- [ ] Images optimised (WebP where possible, max 200kb for non-hero)
- [ ] SSL active
- [ ] Analytics installed and tracking confirmed

---

## D — Print / Collateral Review

- [ ] Files set up in CMYK
- [ ] Bleed added (3mm standard)
- [ ] Fonts embedded or outlined
- [ ] Resolution ≥ 300 DPI for all images
- [ ] PDF/X-1a or PDF/X-4 export ready
- [ ] Spell-checked
